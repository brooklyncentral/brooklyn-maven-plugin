brooklyn:deploy

Full name:

io.brooklyn.maven:brooklyn-maven-plugin:0.1.0:deploy

Description:

Instruct an existing Brooklyn server to deploy the given blueprint.

Attributes:

  • Requires a Maven project to be executed.
  • Binds by default to the lifecycle phase: pre-integration-test.

Required Parameters

Name Type Since Description
blueprint String - The location of the blueprint to deploy. Either a file on disk or a remote URL.
User property is: brooklyn.blueprint.
server URL - The URL of the Brooklyn server to communicate with.
User property is: brooklyn.url.

Optional Parameters

Name Type Since Description
blueprintEncoding String - The encoding of the blueprint to deploy.
Default value is: UTF-8.
User property is: brooklyn.blueprintEncoding.
password String - The password for the user at the Brooklyn server.
User property is: brooklyn.password.
pollPeriod Integer - The period that should be waited between successive polls of the Brooklyn server.
Default value is: 5.
User property is: brooklyn.pollPeriod.
pollUnit TimeUnit - The unit associated with pollPeriod.
Default value is: SECONDS.
User property is: brooklyn.pollUnit.
propertyName String - The property to set to the application id.
Default value is: brooklyn.app.
stopAppOnDeployError boolean - The deploy goal throws a MojoFailureException if waitForRunning is true and the deployed application's status is Status.ERROR. If this property is true the plugin will attempt to stop the application before throwing. If false it will leave the application alive.
Default value is: true.
timeout Integer - The duration mojos should wait for actions at Brooklyn to complete.
Default value is: 5.
User property is: brooklyn.timeout.
timeoutUnit TimeUnit - The unit associated with timeout.
Default value is: MINUTES.
User property is: brooklyn.timeoutUnit.
username String - The user to connect to the Brooklyn server as.
User property is: brooklyn.user.
waitForRunning boolean - Configure the plugin to wait for the deployed blueprint to be running or to throw a MojoFailureException if it is not running within the configured timeout.

If a deployment fails when this property is set then any goals configured in the post-integration-test phase will not run.


Default value is: true.
